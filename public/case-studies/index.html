01

TradingView 信号去重与订单路由

项目背景

客户已有 Pine Script 信号，但重复 Alert 和手工下单导致执行不一致。

约束

不能改客户策略逻辑；必须保留人工暂停；API Key 不包含提现权限。

架构

TradingView Alert -> Webhook 接收 -> 签名校验 -> 去重/冷却 -> 风控 -> 订单路由 -> 告警和日志。

交付物

Webhook 服务、配置样例、风控规则、订单状态日志、Telegram 告警和部署文档。

边界

不评价信号盈利能力，不保证成交价格；验收只看信号解析、拒单、下单和告警链路。

02

券商 API 组合再平衡工作流

项目背景

客户需要按目标权重生成订单计划，并保留人工确认点。

约束

账户权限和可交易品种以券商为准；订单发送前必须展示现金约束和敞口变化。

架构

持仓同步 -> 目标权重计算 -> 订单计划 -> 人工确认 -> 券商 API -> 执行回报 -> 审计日志。

交付物

组合计算脚本、订单计划输出、人工确认节点、日志字段说明和测试样例。

边界

客户确认订单和承担投资决策；系统不提供荐股、调仓建议或收益目标。

03

多接口私有化部署与告警

项目背景

客户需要把信号源、交易 API、日志和告警部署在自己控制的服务器上。

约束

客户保管服务器和密钥；长期运维不包含在初始开发范围；必须支持重启和回滚。

架构

VPS/Docker -> 环境变量 -> 执行服务 -> 日志轮转 -> 告警渠道 -> 备份和恢复说明。

交付物

源码、配置样例、进程守护配置、日志路径、告警配置、部署文档和远程讲解。

边界

交付私有化运行能力，不代管客户账户、服务器、API Key 或日常交易决策。
