What We Build

不是简单写一个下单脚本，而是围绕真实交易流程补齐权限、风控、路由、异常处理和审计链路。

订单路由

限价、市价、分批、撤单、重试、限频、滑点保护和失败告警。

组合自动化

风险控制

后续扩展

Delivery Process

从 API 权限检查到灰度上线

先确认你的券商接口是否能支持目标工作流，再决定开发范围，减少无效投入。

  1. 01接口可行性评估确认券商、账户权限、品种、订单类型和地区限制。
  2. 02规则与风控拆解明确触发条件、仓位约束、异常处理和人工确认点。
  3. 03系统开发与联调搭建订单路由、监控日志、告警和测试环境。
  4. 04模拟盘 / 小资金灰度先验证执行稳定性，再逐步提高自动化比例。
