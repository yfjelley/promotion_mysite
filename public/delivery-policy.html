交付与退款边界

最后更新：2026-06-29。正式项目会先确认需求清单、交付物、验收方式、时间安排和付款节点，避免把策略效果、第三方限制或新增需求混入开发范围。

常见交付物

  • 项目源码、配置样例、部署说明、运行文档和远程讲解。
  • API 接入、订单路由、风控校验、日志审计、告警和必要的测试脚本。
  • 根据项目范围提供模拟盘、小资金灰度或测试环境联调支持。

验收标准

  • 以双方确认的功能清单、接口平台、测试路径和文档交付为验收依据。
  • 系统能按约定规则完成信号解析、风控校验、订单请求、日志记录和告警流程。
  • 盈利结果、胜率、回撤和资金曲线不作为软件项目验收标准。

付款与退款边界

  • 需求初步评估通常不收费；正式开发以双方确认的报价和付款节点为准。
  • 已完成的需求分析、架构设计、开发、联调、文档和讲解，按实际阶段结算。
  • 未开始的后续阶段，如双方确认暂停，可按未发生工作量协商处理。

不包含项目

  • 交易策略盈利承诺、投资建议、账户托管、服务器长期运维和第三方平台费用。
  • 客户账户权限申请、券商或交易所审核、地区合规判断和税务法律意见。
  • 未在需求清单内确认的后台、报表、移动端、低延迟专线或多账户权限系统。

范围变更

如果开发过程中出现新交易规则、新 API 平台、第三方接口变更、账户权限不足或客户新增功能，需要重新评估工时、费用和交付时间。我们会优先建议从最小可验证版本开始，减少无效开发。业务联系邮箱：contact@pddjf.com。
