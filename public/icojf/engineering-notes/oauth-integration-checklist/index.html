OAuth Integration Checklist

OAuth work should be scoped as a production handover problem, not only as a login screen or one successful token exchange.

Before development

  • Confirm the provider, app owner, callback domain, redirect URI, and whether a sandbox account exists.
  • List required scopes and remove permissions that are not needed for the first release.
  • Separate test credentials from production credentials and document who can rotate them.

Token handling

The integration should record token expiry, refresh behavior, error codes, and revoked-access states. A runbook should explain how to reconnect an account without editing source code.

Security boundary

Secrets should live in environment variables or a managed secret store. Logs should keep request IDs and error classes, not raw access tokens or unnecessary personal data.

Acceptance examples

  • First authorization succeeds with the correct redirect URI.
  • Refresh works after access-token expiry.
  • Revoked access creates a clear reconnect state.
  • Wrong scopes fail with a readable error.

Handover

Final delivery should include source code, deployment notes, environment variable names, provider dashboard settings, callback URL list, log fields, and a short recovery guide.
