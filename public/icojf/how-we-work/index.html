How We Work

先把范围、资料、验收和交接讲清楚。

ICOJF Studio 是 Software Product Studio，支持中文/英文远程协作。工作方式是先确认业务目标、现有系统、客户需提供资料、预算、上线窗口和验收负责人，再决定进入原型、接口验证或开发。

Workflow

合作顺序

每一步都产生可检查的交付资产，避免项目只靠口头理解推进。

1

1. 范围评估

确认业务目标、现有系统、负责人、预算范围、上线窗口和第一阶段验收标准。

2

2. 原型或接口验证

把页面流程、数据对象、字段映射、权限、异常路径和客户需提供资料固定下来。

3

3. 分阶段开发

按可验收切片交付，保留演示环境、变更记录、日志说明和样本验收路径。

4

4. 上线和交接

交付源码、部署文档、环境变量、运行手册、验收清单和远程讲解。

Acceptance Boundary

验收证明什么

验收证明约定系统行为能在可复现样本上运行，包括正常路径和常见失败路径。
验收不证明融资、用户增长、广告 ROI、第三方平台审核结果或第三方 API 永久可用。
客户责任提供接口文档、测试账号、样本数据、及时反馈、预算范围和明确验收负责人。
交付资产Source Code（源码）、Deployment Docs（部署文档）、Runbook / runbook（运行手册）、环境变量说明、日志字段和维护建议。
